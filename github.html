GitHub Copilot X

GitHub Copilot X—— 您的 AI 配对程序员

GitHub Copilot 是一个 AI 对程序员，可帮助您更快地编写代码， 减少工作量。它从注释和代码中提取上下文，以立即建议单个行和整个函数。 GitHub Copilot由OpenAI Codex提供支持，OpenAI Codex是由OpenAI创建的生成式预训练语言模型。 它可作为Visual Studio Code，Visual Studio，Neovim和JetBrains集成开发环境（IDE）套件的扩展。


步入新的编程时代

GitHub Copilot由Codex提供支持， Codex是由OpenAI创建的生成预训练AI模型。它已经接受了来自公开来源的自然语言文本和源代码的训练， 包括 GitHub 上公共存储库中的代码。在最近的一项评估中， 我们发现用户平均接受了 GitHub Copilot 显示的所有完成次数的 26%。我们还发现， 平均超过27%的开发人员代码文件是由GitHub Copilot生成的，在某些语言中，如Python，这一比例高达40%。但是， GitHub Copilot 并没有编写完美的代码。它旨在根据它有权访问的上下文生成可能的最佳代码，但它不会测试它建议的代码， 因此代码可能并不总是有效，甚至没有意义。GitHub Copilot 只能保存非常有限的上下文， 因此它可能无法使用项目中其他位置甚至同一文件中定义的有用函数。它可能会暗示库和语言的旧用法或已弃用法。 将用非英语编写的注释转换为代码时， 与英语相比可能存在性能差异。对于建议的代码，与其他编程语言相比，某些语言（如 Python、JavaScript、TypeScript 和 Go）的性能可能更好。 与任何其他代码一样，GitHub Copilot 建议的代码应经过仔细测试、审查和审查。作为开发人员，它随时为您所用。

客户如何充分利用 GitHub Copilot X？

通过使用 GitHub Copilot 并在反馈论坛中分享您的反馈，您可以帮助改进 GitHub Copilot。还请直接向 copilot-safety@github.com 报告事件 （例如，攻击性输出、代码漏洞、代码生成中的明显个人信息）， 以便我们改进保护措施。GitHub 非常重视安全和保障，我们致力于不断改进。
